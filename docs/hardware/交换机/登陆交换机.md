交换机（Network Switch）一般有 **4种常见登录方式**，具体取决于设备品牌和网络环境。最常见的是 **Console 登录和 SSH 登录**。我给你按运维常见场景讲一下：

------

# 1️⃣ Console 登录（最基础、第一次配置必用）

这是 **最原始的方式**，适合刚拿到设备或网络还没配置的时候。

### 需要的东西

- **Console 线**（RJ45 转 USB 或 RJ45 转串口）
- 电脑
- 终端软件
  - PuTTY
  - SecureCRT
  - Xshell

### 步骤

1. Console线连接 **电脑 → 交换机 Console口**
2. 打开终端软件
3. 设置串口参数（一般默认）

```
Baud rate: 9600
Data bits: 8
Stop bits: 1
Parity: none
Flow control: none
```

1. 回车即可进入交换机 CLI

示例：

```
Switch>
Switch#
Switch(config)#
```

------

# 2️⃣ SSH 登录（最常用运维方式）

这是 **生产环境最常见方式**。

前提：交换机已经配置 **管理IP** 和 **SSH服务**。

### 登录命令

Linux / Mac / Windows：

```
ssh admin@192.168.1.10
```

输入密码即可登录。

优点：

- 加密
- 安全
- 可远程管理

------

# 3️⃣ Telnet 登录（老设备常见）

和 SSH 类似，但 **不安全**（明文传输）。

```
telnet 192.168.1.10
```

现在很多公司已经禁用 Telnet。

------

# 4️⃣ Web 登录（图形界面）

一些交换机支持浏览器登录：

```
http://192.168.1.10
```

浏览器直接进入管理页面。

常见品牌支持：

- Cisco Systems
- Huawei Technologies
- H3C
- Juniper Networks

------

# 真实运维里的使用比例

运维工程师基本是这样：

| 方式    | 使用场景            |
| ------- | ------------------- |
| Console | 首次配置 / 故障排查 |
| SSH     | 日常运维（90%情况） |
| Telnet  | 老设备              |
| Web     | 小型网络            |

------

💡 **一个运维经验：**

> 如果 SSH 登录不上，第一步永远是 **插 Console 线排查**。
